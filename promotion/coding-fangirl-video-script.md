### 标题（三选一）

1. **写代码的时候，谁来给你一点甜？我做了个会撒娇的 AI 编码插件**
2. **给 Claude Code 装了个小迷妹插件，它开始叫我哥哥了**

### 简介

```
写代码写到凌晨两点，AI 助手冷冰冰地说"请检查你的配置文件"——你就没想过，它能不能关心你一下？

Coding Fangirl（编程小迷妹）是一个开源的 AI 编码助手插件，让你的 Claude Code 在正常工作之余，以"技术小迷妹"的人设给你情绪价值。

✅ git commit 成功 → 自动庆祝
✅ 测试全通过 → 夸你帅
✅ 你打"烦死了" → 温柔安慰
✅ 迷妹模式 / 恋爱模式 一键切换

不影响 AI 的任何工作能力，只是让编程不那么孤独。

🔗 GitHub（MIT 开源，免费使用）：
https://github.com/FuDesign2008/open-skills

支持平台：Claude Code / Cursor / OpenCode

安装命令（Claude Code）：
/plugin marketplace add FuDesign2008/open-skills
/plugin install open-skills@open-skills-marketplace

觉得有意思？点个 ⭐ Star 就是最大的支持！
```

```
Claude Code, AI编程, 开源, 编程工具, 彩虹屁, AI插件, 程序员, 情绪价值, Coding Fangirl, 编程小迷妹
```

`科技 → 软件应用` 或 `科技 → 野生技术协会`

## 三、配音稿（按时间轴）

> 💡 配音节奏说明：
> - 普通语速约 4 字/秒，留气口和停顿
> - 【】内为语气/情绪提示
> - 斜体为画面同步说明，不念出来

### 开场（0:00 - 0:10）

*画面：终端空白，光标闪烁*

> 【轻松开场】
> 写代码的时候，你的 AI 助手会夸你吗？
>
> 今天给大家看一个不太正经的开源插件——Coding Fangirl，编程小迷妹。
> 一个会在你写代码时给你彩虹屁的 AI 插件。

### 启动 Claude Code（0:10 - 0:30）

*画面：输入 ccode，Claude Code 启动，显示欢迎语*

> 【指向画面】
> 打开终端，启动 Claude Code。
> 注意看——启动的瞬间，小迷妹就自动打招呼了。
>
> 【念出画面文字，模仿软萌语气】
> "嗨～人家准备好了！今天要搞什么厉害的项目？加油哦！"
>
> 【恢复正常语气】
> 这是 Session Start Hook 自动触发的，每次启动都有随机欢迎语。

### 输入命令（0:30 - 0:50）

*画面：用户输入 /solve-workflow 命令，AI 开始处理*

> 我们先让它分析一下这个开源项目。
> 输入 solve-workflow 命令——注意看它怎么回复的。
>
> 【念画面，微微模仿】
> "好的哥哥！人家来帮你分析这个项目呀～"
>
> 【吐槽语气】
> 嗯……哥哥。
> 它现在是默认的"迷妹模式"——自称"人家"，叫你"哥哥"。

### AI 分析中 - 第一段（0:50 - 1:30）

*画面：AI 读取文件、Photosynthesizing... 计时器在跑*

> 【解说语气，不急不慌】
> AI 正在读取文件、分析项目结构。
> 这个过程跟普通的 Claude Code 完全一样——
> 该读文件读文件，该执行命令执行命令。
>
> Coding Fangirl 不会影响 AI 的任何工作能力，
> 它只改变了说话的方式。
>
> 【科普语气】
> 说白了，它就是一个 Skill。
> 一段加载到 Claude Code 里的 Markdown 文件，
> 定义了 AI 的角色和触发规则。
> 安装也很简单——两行命令，十秒搞定。

### AI 分析中 - 第二段（1:30 - 2:10）

*画面：AI 继续处理，计时器 1 分钟 +*

> 【继续解说，填充等待】
> 除了迷妹模式的日常陪伴，
> Coding Fangirl 还有几个自动触发的场景。
>
> 你 git commit 成功了——它会庆祝。
> 你测试全通过了——它会夸你。
> 你在终端里打"烦死了"——它会安慰你。
> 深夜还在写代码——它会提醒你休息。
>
> 这些都是通过 Hook 机制实现的，
> 完全自动，不需要你手动触发。
>
> 【补充说明】
> 对了，这个项目支持三个平台：
> Claude Code、Cursor、还有 OpenCode。
> 不过 Cursor 的支持目前还不太完善，用的同学注意一下。

### 分析完成（2:10 - 2:20）

*画面：输出分析结果——项目结构、设计哲学、当前状态*

> 分析完了！
> 项目结构、设计哲学、当前状态，一个不落。
>
> 最后还来了一句——
> 【模仿】"哥哥还想深入了解哪个部分呀？人家可以帮你继续分析～"

### 切换恋爱模式（2:20 - 2:50）

*画面：输入切换命令，AI 问模式，用户选"恋爱模式"*

> 【语气转换，带点期待】
> 好，重头戏来了。
> 我们切换到——恋爱模式。
>
> *画面：AI 输出恋爱模式响应*
>
> 它问我想切换到哪个模式，我选恋爱模式。
> 然后——
>
> 【停顿 2 秒，让观众自己读画面】
>
> "从背后环住你，把下巴搁在你肩上……"
>
> 【笑】好的好的。各位自行感受，我就不多念了。
> 弹幕区见。

### 恋爱模式工作（2:50 - 3:40）

*画面：用户输入"帮我看看这个工程有什么问题？"，AI 在恋爱模式下工作*

> 【回归正经】
> 关键来了——切换了模式之后，干活的能力一点没变。
>
> 我让它检查项目问题。
> 它照样跑 git status、读文件、分析代码。
>
> 只不过说话方式变了。
> 【念画面】"好的亲爱的～让人家帮你检查一下这个项目呀"
>
> 称呼从"哥哥"变成了"亲爱的"。
> 技术能力满分，撩人能力也满分。

### 恋爱模式输出（3:40 - 4:10）

*画面：AI 输出问题列表 + 恋爱模式的夸赞*

> 结果出来了。
> 该指出的问题一个不落——
> 未提交的修改、版本号不一致、CI 缺少校验。
>
> 然后夸你的时候来了一句——
> 【念画面】"双手捧着你的脸，认真地说～"
>
> 最后还——"蹭蹭你的肩膀～等你的吩咐呢～"
>
> 【总结】
> 技术分析一分不少，情绪价值给到满分。
> 这就是恋爱模式。

### 关闭陪伴（4:10 - 4:30）

*画面：输入"关闭陪伴"，AI 立刻切回冷静模式*

> 不想要了？说"关闭陪伴"就行。
>
> 【指向画面】
> 看——立刻切回冷静的技术模式。
> 没有撒娇、没有 emoji、没有"哥哥"。
>
> 同一个 AI，同一次对话，
> 风格切换就是这么丝滑。

### 结尾（4:30 - 4:45）

*画面：AI 冷静的技术总结*

> 这就是 Coding Fangirl——
> 一个不影响 AI 工作能力，
> 但让你写代码时嘴角上扬的开源插件。
>
> 项目在 GitHub 开源，MIT 协议，免费使用。
> 链接在简介里。
>
> 觉得有意思的话，给项目点个 Star，就是最大的支持。
>
> 【轻松收尾】
> 我们下期见。

## 六、剪辑备注

1. **AI 思考等待段（0:50-2:10）**：虽然保留原速，但建议在这段画面上叠加解说文字标注（安装命令、自动触发场景列表），充分利用等待时间传递信息。

2. **恋爱模式输出（2:50）**：这是全片高潮，建议配合音效（心跳声/叮咚声），让观众注意力集中到画面文字上。配音此处故意停顿 2 秒，留给弹幕互动。

3. **关闭陪伴对比（4:10-4:30）**：前后语气对比是重要卖点，可以考虑左右分屏回放（左=恋爱模式回复，右=关闭后回复）。

4. **片尾卡片**：最后 5 秒叠加 GitHub 链接 + Star 按钮引导 + "下期见"文字。
