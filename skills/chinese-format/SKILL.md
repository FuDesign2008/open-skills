# 中文内容格式规范

> **手动应用**：仅在用户明确要求中文格式检查或格式化时使用，不因普通中文写作、文档生成或编辑自动触发

## 标点符号规则

| 标点 | 使用 | 不使用 | 说明 |
|------|------|--------|------|
| 逗号 | `，` | `,` | - |
| 句号 | `。` | `.` | 句末使用；书名号内的句号可保留原样 |
| 分号 | `；` | `;` | - |
| 冒号 | `：` | `:` | - |
| 问号 | `？` | `?` | - |
| 感叹号 | `！` | `!` | - |
| 括号 | `（）` | `()` | - |
| 书名号 | `《》` | - | 书名号内的英文标点符号可保留（如 《JavaScript: The Good Parts》） |

## 例外情况（保持原文格式）

- **技术术语**：JavaScript、TypeScript、React 等（技术术语与中文标点之间无需空格，直接相邻）
- **代码**：所有代码块、代码片段
- **命令**：git commit、npm install 等
- **URL/文件路径**：网址和路径
- **数据格式**：版本号（8.2.10）、百分比（74%）、数字（3.14）
- **中英文混排**：中英文单词之间可加空格（可选，但技术术语作为整体例外处理）

## 应用范围

| 应用 ✅ | 不应用 ❌ |
|--------|----------|
| 用户明确指定要处理的中文文档或片段 | 未经用户要求的普通中文输出 |
| 用户明确要求检查的中文总结、说明、计划 | 代码本身（字符串除外） |
| 用户明确要求处理的中文注释和文档字符串 | 配置文件内容和代码示例 |

## 注意事项

1. **严格遵循**：所有中文标点必须使用中文标点，不能混用
2. **技术内容保持原文**：代码、命令、路径等技术内容必须保持原文格式
3. **一致性**：确保整个文档的格式保持一致

## 示例对比

### ✅ 正确

```
这是一个示例文档，展示了正确的格式。它包含技术术语如 JavaScript 和 TypeScript。

文件路径：/Users/admin/project/src/index.js
版本号：8.2.10
```

### ❌ 错误

```
这是一个示例文档,展示了错误的格式.它包含技术术语如JavaScript和TypeScript.

文件路径: /Users/admin/project/src/index.js
版本号: 8.2.10
```
