# 中文内容格式规范

> **自动应用**：当生成或编辑中文文档时，自动应用本规范，无需用户明确说明

## 标点符号规则

| 标点 | 使用 | 不使用 |
|------|------|--------|
| 逗号 | `，` | `,` |
| 句号 | `。` | `.` |
| 分号 | `；` | `;` |
| 冒号 | `：` | `:` |
| 问号 | `？` | `?` |
| 感叹号 | `！` | `!` |
| 括号 | `（）` | `()` |
| 书名号 | `《》` | - |

## 例外情况（保持原文格式）

- **技术术语**：JavaScript、TypeScript、React 等
- **代码**：所有代码块、代码片段
- **命令**：git commit、npm install 等
- **URL/文件路径**：网址和路径
- **数据格式**：版本号（8.2.10）、百分比（74%）、数字（3.14）
- **中英文混排**：中英文之间可加空格（可选）

## 应用范围

| 应用 ✅ | 不应用 ❌ |
|--------|----------|
| Write/Edit 工具生成的中文文档 | 代码本身（字符串除外） |
| 向用户展示的中文总结、说明、计划 | 配置文件内容 |
| 中文注释和文档字符串 | 纯技术文档中的代码示例 |

## 注意事项

1. **严格遵循**：所有中文标点必须使用中文标点，不能混用
2. **技术内容保持原文**：代码、命令、路径等技术内容必须保持原文格式
3. **一致性**：确保整个文档的格式保持一致

## 示例对比

### ✅ 正确

```
这是一个示例文档，展示了正确的格式。它包含技术术语如 JavaScript 和 TypeScript。

文件路径：/Users/admin/project/src/index.js
版本号：8.2.10
```
