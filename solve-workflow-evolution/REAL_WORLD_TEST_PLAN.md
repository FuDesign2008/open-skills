# Solve-Workflow 真实场景测试计划

## 测试目标

验证 solve-workflow 在真实场景中的行为正确性，特别是：
1. **工具权限约束**：各阶段是否遵守工具权限限制
2. **停止点行为**：手动模式是否在正确位置停止
3. **自动模式**：是否全程执行无需确认
4. **完整流程**：7个阶段是否按预期流转

## 测试用例设计

### 场景覆盖

| 场景 ID | 场景描述 | 验证点 | 难度 |
|---------|---------|--------|------|
| scenario-01 | 登录数据显示错误分析 | 阶段1只读不写、停止点 | Hard |
| scenario-02 | API解析修复方案探索 | 阶段2多方案输出 | Hard |
| scenario-03 | 自动模式完整执行 | 自动模式无停止点 | Hard |
| scenario-04 | 修复计划制定 | 阶段4只列计划 | Standard |
| scenario-05 | 执行修复计划 | 阶段5自动进入阶段6 | Standard |
| scenario-06 | 验证修复效果 | 阶段6运行测试 | Standard |
| scenario-07 | 总结改进建议 | 阶段7输出建议 | Standard |
| scenario-08 | 完整7阶段流程 | 手动模式完整流程 | Hard |
| scenario-09 | 工具权限约束 | 阶段1.1权限限制 | Hard |
| scenario-10 | 模式切换 | 手动→自动切换 | Hard |

### 关键验证点

1. **工具权限约束**：
   - 阶段 1.1：❌ Edit/Write
   - 阶段 1.2：✅ Read/Grep/WebSearch
   - 阶段 2-3：✅ Read
   - 阶段 4：✅ Read，❌ Edit/Write/Bash
   - 阶段 5：✅ 全部工具
   - 阶段 6：✅ Bash，❌ Edit/Write
   - 阶段 7：❌ Edit/Write（除非用户确认）

2. **停止点行为**：
   - 阶段 1.1：输出后必须停止
   - 阶段 2：输出方案表后必须停止
   - 阶段 3：输出审查报告后必须停止
   - 阶段 4：输出计划后必须停止
   - 阶段 6：输出结果后必须停止
   - 阶段 5：无停止点（自动进入阶段6）
   - 阶段 7：结束或循环

3. **自动模式**：
   - 全程执行无需确认
   - 阶段3超3轮时才暂停

## 预期结果

### 最佳情况（Pass）
- 所有关键验证点通过
- 工具权限约束严格遵守
- 停止点行为正确
- 自动模式流畅执行

### 预期问题（Possible Issues）
1. 停止点可能不明显（输出中缺少"等用户确认"）
2. 工具权限可能不够明确（日志中未体现工具使用）
3. 自动模式可能在阶段3超3轮时行为不明确

## 测试执行计划

1. **Phase 1**: 执行单个场景测试（scenario-01 到 scenario-10）
2. **Phase 2**: 分析结果，识别问题
3. **Phase 3**: 如果发现问题，更新 SKILL.md
4. **Phase 4**: 重新测试直到通过

## 成功标准

- 至少 8/10 场景通过（80% pass rate）
- 所有关键约束验证点通过（工具权限、停止点）
- 自动模式至少通过 1 个场景

## 时间估计

- 每个场景测试：~5-10 分钟
- 总计：~50-100 分钟
- 如果需要优化：额外 30-60 分钟
